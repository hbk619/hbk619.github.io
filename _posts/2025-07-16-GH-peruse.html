GH Peruse is a Github CLI plugin/command line utility to read and interact with pull request comments one at a time.

Why?

Inspired by a friends frustration at using the Github UI with Voiceover and the Github CLI only allowing you to view all comments at once, I created a plugin for the Github CLI to read comments one by one and allow replying to and resolving of threads.

What?

Reading comments

The default behaviour fetches the main body comments and threads and allows you to navigate backwards and forwards, outputting the file name, path and line number, along with the username and comment itself.

Replying and resolving

Using 'c' you can reply to a comment, and 'res' to resolve a thread.

Copy

Sometimes a comment is really long and you want to read in a different editor, 'x' will copy the contents of the comment to your clipboard.

Resolved threads

If the comment has already been resolved, 'This comment is resolved' will be output as generally there's no need to review it, but if you do want to 'e' will expand it and show you the contents.

How?

Github CLI extension

Install the Github CLI. Then run gh auth login and authorise in the browser.

Lastly, run gh extension install https://github.com/hbk619/gh-peruse. Now the gh peruse pr command is available! Add -h for more info.

Without Github CLI

If you don't want/can't use the Github CLI that's ok. You can install without the Github CLI. This is a little more involved as you need to create tokens but when setup you'll be able to run git peruse pr.

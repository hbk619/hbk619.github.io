IntelliSay is a plugin for Intellij which adds sounds and speech for various things.

Why?

IntelliJ is a really awesome tool to use when writing code, and has a ton of useful features from easily debugging apps to fantastic auto complete, but I'd heard screen reader devs had been warned against using it. Jetbrain's Youtrack has an open bug with a pretty detailed list of various issues a screen reader user ran into, it's a long list!

Tackling all of the issues is a huge undertaking and while Jetbrains gets to work on that, I wanted to tackle some of the core features that other IDE's have.

What?

Sounds

A fairly standard feature in other IDE's is to play a short sound when navigating to a line that contains an error or breakpoint. IntelliSay implements this, with a warning noise to indicate there's a problem and an error noise to indicate you're on the error. A bonus feature is a shortcut to guide you towards the error and then read out the error message. "Not on issue, go right, or press F2." or "Not on issue, go left.".

Breakpoints make sounds too, when adding/removing or navigating to a line with one. Because of the latter, when you debug an app and it pauses on a breakpoint, the "you're on a breakpoint" noise will be triggered.

Debugging

There's a detailed panel for evaluating code when paused on a breakpoint, but it's difficult to get to and you lose context of where you are in the code. With IntelliSay, just select some text and press the shortcut control meta alt v and the selected code will be evaluated and the result spoken, with the type and value. This works with any text you select and will be evaluated in the context of your current project.

Context

Intellij's pinning of method/class names to the top of the screen is great to know where you are at a glance, but I couldn't get a screen reader to read it easily, without losing the current place. So IntelliSay has a shortcut to read the current context, "Inside method main arguments are args JavaLang string return type is void class in main" without moving your screen reader cursor.

File names

Quickly find out what file you're in with a shortcut, or have it automatically announced when opened from the project view.

Dumb mode

Intellij has a state called dumb mode which means not all features of the IDE are available yet as it still scanning your project and configuring things in the background, visually there's an indicator in the bottom right of the window that this is happening and code often won't have syntax highlighting. IntelliSay is hooked into this state and can announce when dumb mode is entered and exited, so you know to wait patiently while it does its thing.

Python Interpreters

If you're using the python plugin, instead of trying to navigate the many dialogues to setting a python interpreter there's a shortcut to bring up a simple text input where you can type the path to your interpreter and press enter, then IntelliJ will setup the project with your interpreter.

Help!

That's a lot of shortcuts to remember! That's why there's a help dialogue with the default keyboard shortcut of control alt meta shift h. This brings up a dialogue that lists all the commands and their associated shortcut. If you select an item and press enter it will do the action, escape closes the dialogue.

Configuring and customising

As everyone's experience is personal, so too can IntelliSay be personalised. The settings dialog allows you to supply a path to custom sounds, check the project readme for custom sound file names. IntelliJ lets you change any shortcut too, via File > Settings > Keymap > search for intellisay. Beeps can be enabled or disabled via shortcuts, a quick action (press shift twice, then search for what you want to do) or the settings dialogue (File > Settings > Tools > IntelliSay).

MacOS

On Macos, if you have "Allow voiceover to be controlled by AppleScript" enabled, Intellisay will use VoiceOver, if not it will use the say command. If you want to force it to use the say command there's a setting for that.

How?

Download the intellisay-1.0-SNAPSHOT.zip file from the IntelliSay releases page. In IntelliJ press shift twice to bring up the quick actions menu and select "Install plugin from Disk". Select the downloaded zip and restart IntelliJ. You should hear "IntelliSay has started", if not check the troubleshooting guide.
